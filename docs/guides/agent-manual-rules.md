# Agent 手册维护规则

该规则用于维护文档：`docs/guides/agent-cli-quick-manual.md`。

当你要让 agent 修改这份手册时，只需要把本文件提供给 agent，并要求其严格遵守。

## 1. 目标

输出一份“短、小、准”的 agent 操作手册：
- 面向执行，不做背景科普
- 仅保留高频命令
- 默认 JSON 工作流

## 2. 内容边界（必须）

必须包含：
- 前置检查（`dida365-cli auth status`）
- 高频命令：
  - `projects list`
  - `projects create`
  - `list`
  - `show`
  - `add`
  - `update`
  - `done`
  - `delete`
- 执行策略：
  - 读操作默认 `--json`
  - 写操作先 `--dry-run`
  - 常见失败处理（401/403/404）

必须避免：
- 长篇 OAuth 解释
- 低频命令堆叠
- 与当前 CLI 不一致的参数
- 不可执行的伪命令

## 3. 格式约束

- 标题简短，段落短句。
- 命令必须可复制执行。
- 只展示当前仓库实际存在的 flag。
- 示例优先使用 `<project_id>` / `<task_id>` 占位符。
- 若输出面向机器，优先 `--json`。

## 4. 更新流程

每次修改手册时，必须执行：
1. 先运行命令帮助核对参数：`dida365-cli <cmd> --help`
2. 更新手册命令与说明
3. 若命令行为变化明显，同步更新：
   - `README.md`
   - `doc/man/dida.1.md`

## 5. 验收标准

- 手册中的每个命令都可在本仓库当前版本执行。
- 手册正文尽量控制在 1 页内（精简优先）。
- 不需要人工二次解释即可让 agent开始执行任务。
