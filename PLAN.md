# dida365-cli 执行计划

## 1. 目标与范围
- 目标：实现一个可用的 Dida365 命令行工具，命令名为 `dida`。
- 技术约束：使用 Golang 开发。
- 体验约束：参考 `things-cli`（`~/workspace/sources/things3-cli`）的命令结构、参数风格和核心使用路径。
- 接口约束：基于 Dida365 OpenAPI（https://developer.dida365.com/docs#/openapi）实现。

## 2. 关键输入信息
- 已知外部依赖：
  - Dida365 OpenAPI 文档（认证方式、资源模型、接口能力）
  - `things-cli` 源码（命令组织、参数约定、输出方式）
- 当前仓库状态：已完成 MVP 工程骨架与核心命令实现，正在补齐测试、错误处理与使用体验。

## 3. 执行阶段

### 阶段 A：调研与规格固化
- 任务：
  - 梳理 Dida365 API 的认证流程、任务列表查询、任务创建/更新/完成/删除等关键接口。
  - 梳理 `things-cli` 的命令设计，抽取可复用交互模式（命令层级、过滤参数、输出格式）。
  - 梳理 `things-cli` 中面向 agent/协作的文档（如 AGENTS.md、流程说明、约定文件）：统计有哪些文档、各自作用、信息组织方式，并提炼可迁移到本项目的文档结构。
  - 专项调研 `things-cli` 的认证方式（认证流程、凭证存储、刷新机制、异常处理），并映射到 Dida365 场景。
  - 全量盘点 `things-cli` 命令行能力（命令树、参数、输出、典型工作流），形成能力基线。
  - 调研 `things-cli` 的存储设计（SQLite 依赖、表结构/数据模型、读写策略、缓存与同步边界），评估对 `dida` 的可借鉴实现。
  - 形成 MVP 命令清单与参数规范。
- 产出：
  - 命令与参数规格草案（MVP）
  - API 映射表（命令 -> Dida365 接口）
  - `docs/research/things-agent-docs.md`：`things-cli` agent 相关文档清单与结构借鉴建议
  - `docs/research/things-auth.md`：`things-cli` 认证方式调研与 Dida365 适配方案
  - `docs/research/things-cli-capabilities.md`：`things-cli` 完整命令能力说明
  - `docs/research/things-storage.md`：`things-cli` 存储设计（SQLite）调研与 `dida` 存储方案建议

### 阶段 B：工程骨架搭建
- 任务：
  - 初始化 Go 模块与目录结构（`cmd/`, `internal/` 等）。
  - 选择并接入 CLI 框架（优先 `cobra`）。
  - 建立统一配置体系（例如 token、默认项目、输出格式），支持环境变量与配置文件。
- 产出：
  - 可编译运行的 `dida` 基础命令
  - 配置加载与基础日志/错误处理机制

### 阶段 C：认证与 API 客户端
- 任务：
  - 封装 Dida365 API 客户端（HTTP、鉴权、重试、错误解析）。
  - 实现登录/授权初始化命令（若 API 模式需要 OAuth，则提供授权指引与 token 持久化）。
  - 增加接口层单元测试（使用 mock HTTP）。
- 产出：
  - 稳定可复用的 API client
  - 可验证的认证流程

### 阶段 D：MVP 命令实现
- 任务：
  - 实现核心任务命令（建议优先）：
    - `dida list`：任务列表与过滤
    - `dida add`：创建任务
    - `dida show`：查看任务详情
    - `dida done`：完成任务
    - `dida delete`：删除任务
  - 输出支持表格/纯文本（后续可扩展 JSON）。
  - 参数与帮助信息对齐 `things-cli` 的易用性风格。
- 产出：
  - MVP 可用命令集
  - 基础使用文档

### 阶段 E：质量与发布准备
- 任务：
  - 补充单元测试与关键集成测试（命令级）。
  - 完善错误提示、退出码、边界场景（网络失败、鉴权失效、参数冲突）。
  - 增加 `README`（安装、配置、示例）与版本信息命令。
  - 提供构建脚本（本地 + CI 可选）。
- 产出：
  - 可发布的初版 CLI
  - 明确的验收结果与后续迭代入口

## 4. MVP 功能边界（首版）
- 必做：认证、任务增删改查核心路径、清晰帮助信息、可读输出。
- 暂缓：高级视图定制、复杂批量操作、插件化扩展。

## 5. 验收标准
- 可通过 `go build` 生成 `dida` 可执行文件。
- 用户可完成“认证 -> 查看任务 -> 创建任务 -> 完成任务 -> 删除任务”全流程。
- 常见错误场景有可理解提示和非 0 退出码。
- 主要命令具备 `--help` 与示例。

## 6. 风险与应对
- 风险：Dida365 认证流程复杂或存在调用限制。
  - 应对：优先打通认证最小路径，保留 token 刷新与降级策略。
- 风险：`things-cli` 与 Dida365 领域模型不完全一致。
  - 应对：保持交互风格一致，语义不强行一一映射，必要时在帮助文档中标注差异。
- 风险：API 字段/行为与文档有偏差。
  - 应对：通过真实请求验证关键接口，并在 client 层集中兼容。
